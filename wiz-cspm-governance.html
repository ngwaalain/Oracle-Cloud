End-to-end logical flow

All steps with owners and SLAs

0
Pre-assessment — org control / rule review Cloud Security + Arch. Security
One-time framework gate. Both teams jointly review all CSPM controls to confirm they are valid, necessary, and appropriate. Controls identified as inapplicable or duplicative are removed from the framework before the exception process begins. This prevents spurious exceptions down the line.
⏱ Periodic review — not per-exception
A
Path A — App owner deploys resource App owner
Resource deployed via GitHub or IaC pipeline. Wiz CI/CD scanner (with IaC prevent policy configured inside Wiz) detects misconfiguration at build or runtime and raises a Wiz finding.
⏱ Automated detection — no SLA
B
Path B — App owner requests control exception App owner
App owner identifies a CSPM control that is not needed or applicable for their platform or specific resources. They request a control-level exception directly — no Wiz finding is required to trigger this path.
⏱ Owner initiates — no automated trigger
1
App owner submits proposed exception App owner
Owner submits: resource details, business justification, and deviation type — Exception (time-limited), Design, False Positive, or Risk Accepted. Deviation type determines which approval path is followed.
⏱ Owner has 2 business days to submit after finding
2
Platform control review Cloud Security
Cloud Security validates the technical context of the exception request. Confirms the resource details are accurate and the control applies. Escalates to finding review if context is unclear.
⏱ SLA: 2 business days
3
Finding review Arch. Security
Arch. Security reviews the Wiz finding or control exception in context of the submitted deviation type. Confirms scope and validates the justification before risk scoring.
⏱ SLA: 2 business days
4
CargillNow exception ticket created System
Exception ticket opened in CargillNow / ServiceNow. Deviation type is recorded and routes the ticket to the appropriate approval workflow and approval weight required.
⏱ Same day as step 3 completion
5
Risk assessment Arch. Security + Cloud Security
Both teams jointly score five dimensions: Exposure (0–25), Data sensitivity (0–25), Identity blast radius (0–20), Attack path length (0–15), Criticality (0–15). Total score out of 100 gates the approval path. Score <30 → expedited. 30–69 → standard. ≥70 → CISO escalation required.
⏱ SLA: 3 business days
6a
Approved → notify System / Cloud Security
Approval recorded in CargillNow. ServiceNow auto-notifies app owner and resource owner. Email sent as secondary channel only.
⏱ Automated within 1 hour of approval decision
7
Implement approved exception Cloud Security
Cloud Security implements via Wiz ignore policy update or IaC Git PR. PR follows the formal gate process (see IaC PR gate tab): linked to CargillNow ticket, 1 reviewer, Wiz scan must pass, expiry date in metadata.
⏱ SLA: 2 business days
8
Expiry enforcement ServiceNow automated
ServiceNow tracks expiry date set at approval. T-30 days: warning to owner. T-7 days: manager escalation if no renewal action. T+0: Wiz ignore policy auto-revoked, resource re-enters findings queue, CargillNow ticket closed as expired.
⏱ Fully automated — no manual SLA needed
6b
Rejected → remediation loop Cloud Security / Arch. Security
Rejection recorded in CargillNow with reason code and remediation options: (1) fix the resource, (2) re-submit with stronger justification, or (3) escalate to CISO for risk acceptance sign-off. Owner must respond within 5 business days or manager is auto-notified and resource flagged non-compliant in Wiz.
⏱ Owner SLA: 5 business days to respond

Deviation type — approval routing

False Positive
Lightweight path — the control flagged something that is not actually a risk. Technical validation only, no risk committee needed.
Platform engineer (confirms FP) Cloud Security sign-off
⏱ Total SLA: 2 business days
Design exception
Architectural review required — the control conflicts with an intentional design pattern. May recur across multiple resources so the pattern must be assessed, not just the instance.
Platform control review Arch. Security App owner sign-off
⏱ Total SLA: 5 business days
Exception (time-limited)
Standard path with mandatory expiry. Control is valid but a temporary deviation is acceptable with justification. Renewal requires full re-justification — not auto-renewed.
Platform control review Arch. Security Risk score gate
⏱ Total SLA: 5 business days · Max duration: 90 days
Risk Accepted
Heaviest approval path — permanent or long-term acceptance of a known risk. Requires executive sign-off and mandatory annual review. Risk score ≥70 means CISO sign-off is required regardless of deviation type.
Platform control review Arch. Security Risk score gate CISO sign-off required Annual review mandatory
⏱ Total SLA: 10 business days · Annual renewal

Rejection and remediation flow

R0
Approval denied Arch. Security / Cloud Security
Risk assessment results in a denial. Decision recorded in CargillNow with the approving team, risk score, and specific reason for rejection.
⏱ Rejection notification automated within 1 hour
R1
Rejection issued with guidance Cloud Security
CargillNow ticket updated with: reason code, specific control or policy violated, and three remediation options presented to the owner: (1) fix the misconfiguration and close, (2) re-submit with stronger justification, (3) escalate to CISO for risk acceptance sign-off.
⏱ Owner must respond within 5 business days
R2
Owner response App owner
If owner fixes the resource: Wiz scanner validates automatically and finding closes. If re-submitting: exception re-enters the flow at step 1 with prior rejection history pre-populated in CargillNow. If escalating: CISO review triggered as an addition to the standard risk assessment path.
⏱ Response deadline: day 5 from rejection
R3
No response — escalation Automated
If no owner response by day 5: manager automatically added to CargillNow ticket, resource tagged as non-compliant in Wiz, and finding escalated to CISO dashboard. Resource remains non-compliant until resolved.
⏱ Triggered automatically at day 5

Auto-expiry lifecycle — ServiceNow managed

Day 0 — exception approved and implemented
Expiry date set in CargillNow record. Wiz ignore policy applied with expiry metadata embedded. ServiceNow creates an auto-task for expiry tracking — no manual calendar reminders needed.
T-30 days — first warning
ServiceNow notifies resource owner and app owner via CargillNow ticket. Owner is prompted to review and either initiate renewal (re-enters flow at step 1 with history pre-populated) or confirm the exception is no longer needed and close it.
T-7 days — escalation if no renewal action
If no renewal or closure action has been taken: manager automatically added to the CargillNow ticket. Email sent as secondary channel. Exception flagged on CISO dashboard as "expiring — no action."
T+0 — auto-revocation
If still no response: Wiz ignore policy automatically removed. Resource re-enters the Wiz findings queue. Compliance report updated. CargillNow ticket closed as "expired — not renewed." IaC metadata expiry hook triggers the PR/policy removal.
Renewal path (any time before T+0)
Owner triggers renewal from CargillNow at any point before expiry. Exception re-enters the workflow at step 1 with full prior history pre-populated. Risk score is re-evaluated — score may gate a higher approval weight than the original exception required.

Formal IaC PR / merge gate

PR creation
→
PR opened by Cloud Security, title includes CargillNow ticket ID
→
Branch naming: exception/CNOW-{ticket-id}-{deviation-type}
→
PR description auto-includes: deviation type, risk score, approver names, and expiry date
→
Replaces the informal dotted-line connection between IaC Git and Wiz CI/CD scanner — all changes go through this gate
Required checks before merge
✓
Wiz CI/CD scanner must pass on the PR diff — no new violations introduced by the exception change
✓
Minimum 1 reviewer approved (must be Cloud Security or Arch. Security team member)
✓
CargillNow ticket status = "Approved" — verified by CI pipeline via API before merge is permitted
✓
Expiry date present in IaC metadata — enforced by pre-merge hook, merge blocked if missing
Audit trail
✓
Merge commit SHA written back to CargillNow ticket for end-to-end traceability
✓
All reviewer comments and approval timestamps retained in Git history
✓
Wiz scanner result artifact attached to PR for audit and compliance evidence
✓
Auto-revocation hook at T+0 reads expiry from IaC metadata and triggers policy removal PR automatically

Notification channels — ServiceNow as single source of truth

🎫
CargillNow / ServiceNow
All exception lifecycle events — created, approved, rejected, expiring, expired, renewed. Audit log, ticket history, SLA tracking, and renewal triggers all live here. This is the system of record.
Primary
💬
Teams / Slack (via ServiceNow webhook)
Key events pushed to Cloud Security and app owner channels. Rejection notices and T-7 expiry warnings get @mentions. Configured as a ServiceNow outbound webhook — no manual notification steps.
Secondary
📧
Email
Sent as CC / backup only via ServiceNow email action. Not the system of record. No manual email steps exist anywhere in the process — email is a redundancy layer only.
Demoted
📊
CISO dashboard
Real-time view of all open exceptions by risk score, deviation type, days-to-expiry, and team. Populated automatically from ServiceNow + Wiz API. High-risk and expiring-soon exceptions are surfaced automatically.
